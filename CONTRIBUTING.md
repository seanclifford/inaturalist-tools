# Contributing

## Before You Start

This is a personal project maintained in my limited spare time. I am happy to discuss ideas, bug reports, and feature requests through GitHub Issues. Responses may be slow.

### Pull Requests Require Approval as a Collaborator

If you would like to contribute code:

1. Send me a message on iNaturalist or the iNaturalist Forums.
2. Let me know how you'd like to contribute.
3. If it sounds good to me, I'll add you as a collaborator and you can then make Pull Requests.

NOTE: any large contributions should be discussed beforehand to prevent you wasting your time on something I'm not happy to merge.

## Reporting Issues

Bug reports, questions, and feature suggestions are welcome.

When creating an issue, please:

* Clearly describe the problem or suggestion.
* Include relevant details, examples, logs, or screenshots where appropriate.
* Check existing issues before creating a new one.

### AI-Generated Issues

Issues created primarily by AI tools without verification, testing, or personal understanding are not welcome.

If you use AI assistance while preparing an issue, you are responsible for ensuring that:

* The issue describes a real problem or genuine enhancement.
* The information is accurate.
* The issue reflects your own investigation and understanding.

Low-quality AI-generated issue submissions may be closed without further discussion.

## Code of Conduct

Please be respectful and constructive in all interactions.

## Questions

If you're unsure whether a contribution would be useful, open an issue and ask first.
